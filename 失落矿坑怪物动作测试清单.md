# 《凡戴尔的失落矿坑》怪物动作建模测试清单

本文档整理当前 `E:\TRPG-Agent` 中已经接入的 Lost Mine 怪物结构化动作，方便后续人工跑团/自动化测试时逐项验收。

## 数据来源说明

当前怪物数据是混合来源：

- 基础模板来自 Open5e：`AC`、`HP`、速度、六维属性、CR、普通攻击、抗性/免疫/易伤。
- Lost Mine 关键怪物动作来自本地结构化表：`backend/app/monsters/lost_mine.py`。
- 生成怪物时会先拉 Open5e 模板，再合并本地结构化动作。若某个模组专名 slug 不存在于 Open5e，后续还需要补本地模板或 slug 映射。

当前本地结构化动作表覆盖 `18` 个 slug。

## 需要人工重点复测的场景

1. `spawn_monsters` 对模组专名 slug 的可用性

   本地动作表已经有 `redbrand-ruffian`、`evil-mage`、`nezznar` 等 slug，但 `spawn_combatants()` 仍先请求 Open5e。如果 Open5e 没有对应 slug，需要补本地模板 fallback 或 slug 映射。

2. 怪物反制玩家施法

   机制已经接入，但当前 Lost Mine 本地动作数据没有给具体怪物配置 `counterspell` reaction。测试中是手工构造带 `counterspell` 的 Evil Mage 来验证机制。

3. `spell_reflection`

   Spectator 的四种射线已经建模；`spell_reflection` 仍未实现为 reaction hook。

4. 施法怪 AI

   当前仍由 LLM 根据 combat context 中展示的 `actions` 选择动作；系统负责规则结算。还没有独立战术选择器。

5. Open5e 防御字段的细粒度限制

   抗性/免疫/易伤已进入统一伤害管线，但类似 “非魔法武器造成的钝击/穿刺/挥砍抗性” 暂时会简化成对应伤害类型抗性，尚未区分魔法/非魔法来源。

## 手工验收建议顺序

1. Goblin / Wolf / Ghoul：确认普通攻击、命中后豁免和状态生命周期。
2. Giant Spider / Young Green Dragon：确认多段伤害、范围、recharge。
3. Redbrand Ruffian / Owlbear / Grick：确认多重攻击动作经济。
4. Bugbear / Hobgoblin / Zombie / Wraith：确认 trait hook。
5. Evil Mage / Flameskull / Nezznar：确认怪物施法、专注、反应、空间点选。
6. Spectator：逐个测试四射线，再测 `eye_rays`。
7. 最后测试 Open5e 回退单位，记录哪些 slug 需要本地模板 fallback。
